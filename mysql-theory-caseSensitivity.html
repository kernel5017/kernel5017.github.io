mysql-theory-caseSensitivity

Created2020-12-23|Updated2020-12-23|mysql
|Post View:

​ 在mysql中，一个database就是datadir目录下的一个文件夹。这个database下的表在这个文件夹下会有一个或多个文件。且mysql的trigger也是以文本的形式记录在相应的文件夹下，以.TRG结尾，可以使用cat命令查看。所以对于大小写敏感问题，还取决于当前数据库处于的系统是否是大小写敏感。也就是说，这些文件夹和文件在windows系统下是不区分大小写，但是在大多数unix系统下是区分大写偶写的。系统参数lower_case_table_names控制着大小写问题。

​ column,index,stored routine,event name,column aliases这些在任何平台都是不区分大小写的，你可以随便用大写小写都可以。

​ logfile groups是区分大小写的。

​ 同样的语句在win下就是可以的，所以为了避免产生不必要的麻烦，尽量约定都使用同一的小写字符作为数据库和表的名称。

​ 系统参数lower_case_table_names 控制着database和table的名字如何在disk上存储，如果和mysql中使用的问题。这个参数不糊影响trigger的大小写。在unix系统下，这个参数的默认值为0，在win下为1，在os x下为2。

value 0

使用create table或create database时候，创建的表或者数据库使用指定的字母，也就是区分大小写，你使用大写那么在磁盘上存储的就是大写的。在不分区大小写的系统上比如win上，不应该设置为0.如果在不区分大小写的系统上强制设置这个值为0，那么使用不同的大小写字母访问myisam表，可能导致索引损坏。

value 1

table name使用小写存储在磁盘上，名称比较不区分大小写。mysql会将素有表名转换为小写字母进行存储和查询。对于database name和table aliases也是一样的效果。也就是你create table TeST,在存储是会存储为test;你select TEST，会转换为select test; insert TEST会转换为insert test。

value 2

使用create table or create database时候使用指定的大小写字符，但是在进行lookup的时候，mysql会将所有字符转换为小写。对于名称比较是不区分大小写的。这个只对于不区分大小写的系统来说。INNODB tables还是使用小写字母进行存储。

如果你使用的mysql是单一的系统，不会从一个平台迁移到另一个平台，比如从linux换到windows，那么就不用考虑大小写问题。比如在linux中你有表test和TEST，如果迁移到win中就会出现问题，因为对于win来说这两个表是相同的。

如果确实存在需要迁移的情况，那么有两个选择：

  • 在所有系统上设置lower_case_table_names=1。缺点就是使用show tables或者show databases时候，不会看到原本的表名，看到的都是小写的。
  • 在unix上设置为0，在win上设置为2。

Exception: If you are using InnoDB tables and you are trying to avoid these data transfer problems,
you should set lower_case_table_names to 1 on all platforms to force names to be converted to
lowercase (p1455)

如果你想在unix系统上将这个值设置为1，那么你首先必须在重启系统前（设置此参数需要重启系统，在配置文件中配置）将所有表名和数据库名改为小写的！

如果想转换一个或者多个database，在设置lower_case_table_names之前使用mysqldump备份所有数据库，然后drop这些数据库，然后设置参数为1，然后重启。具体步骤如下：

1.dump database

2.使用drop database删除每个库

3.stop 数据库，设置lower_case_table_name，然后启动数据库

4.reload dump文件，因为lower_case_table_names已经设置了，所以这时reload的时候都会转换为小写的
